Python中的os模块

Posted on 2019-03-14 | Post modified: 2019-03-14

os模块的应用非常广泛，提供了非常丰富的方法用来处理文件和目录。比如在自动化测试中，经常需要查找操作文件，比如说查找配置文件（从而读取配置文件的信息），查找测试报告（从而发送测试报告邮件），经常要对大量文件和大量路径进行操作，这就依赖于os模块。
下面开始整理其使用方法：

os.access(pass,mode)

检验权限模式

os.access()方法使用当前的uid/gid尝试访问路径。大部分操作使用有效的uid/gid, 因此运行环境可以在 suid/sgid 环境尝试。

参数

  1. path——>用来检测是否有访问权限的路径。
  2. mode——>测试存在的路径：
    • os.F_OK:包含在access()的mode参数中，测试path是否存在。
    • os.R_OK:包含在access()的mode参数中，测试path是否可读。
    • os.W_OK:包含在access()的mode参数中，测试path是否可写。
    • os.X_OK:包含在access()的mode参数中，测试path是否可执行。

返回值

如果允许访问返回True，否则返回False。

例子

os.chdir(path)

改变当前工作目录

os.chdir() 方法用于改变当前工作目录到指定的路径。

参数

path——>要切换到的新路径。

返回值

如果允许访问返回True，否则返回False。

例子

os.chflags(path,flags)

（只支持在Unix下使用，这里只说一下它的使用方法，先码住说不定有用呢）
设置路径的标记为数字标记

os.chflags() 方法用于设置路径的标记为数字标记。多个标记可以使用OR来组合起来。

参数

  1. path——>文件名路径或目录路径。
  2. flags——>可以是以下值
    • stat.UF_NODUMP: 非转储文件
    • stat.UF_IMMUTABLE: 文件是只读的
    • stat.UF_APPEND: 文件只能追加内容
    • stat.UF_NOUNLINK: 文件不可删除
    • stat.UF_OPAQUE: 目录不透明，需要通过联合堆栈查看
    • stat.SF_ARCHIVED: 可存档文件(超级用户可设)
    • stat.SF_IMMUTABLE: 文件是只读的(超级用户可设)
    • stat.SF_APPEND: 文件只能追加内容(超级用户可设)
    • stat.SF_NOUNLINK: 文件不可删除(超级用户可设)
    • stat.SF_SNAPSHOT: 快照文件(超级用户可设)

返回值

无
