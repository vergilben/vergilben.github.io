开始使用Burpsuite

Burpsuite是用于攻击web应用程序的集成平台，包含了许多工具。Burpsuite为这些工具设计了许多接口，以加快攻击应用程序的过程。所有工具都共享一个请求，并能处理对应的HTTP消息、持久性、认证、代理、日志、警报。当Burpsuite 运行后，Burp Proxy开启默认的8080端口作为本地代理接口。通过置一个web浏览器使用其代理服务器，所有的网站流量可以被拦截，查看和修改。默认情况下，对非媒体资源的请求将被拦截并显示(可以通过Burp Proxy 选项里的options选项修改默认值)。对所有通过Burp Proxy 网站流量使用预设的方案进行分析，然后纳入到目标站点地图中，来勾勒出一张包含访问的应用程序的内容和功能的画面。在Burp Suite 专业版中，默认情况下，Burp Scanner是被动地分析所有的请求来确定一系列的安全漏洞。

一、环境配置

Burpsuite的运行需要java环境。这里要特别说明一下，专业版的Burpsuite需要用keygen来使用，所以一定要安装JRE，前面我在尝试的时候安装了JDK和java都无法运行，安装过JRE后才可以，后面会说到这里的。这里附上最新版本的JRE下载链接：https://pan.baidu.com/s/1q90fsZlcEHZVHsB8HJnqTQ 提取码：6jhn
安装完成后在cmd下输入：java -version，如果返回了版本号说明安装成功：

需要注意的是，打开Burpsuite必须要先打开burp-loader-keygen.jar，再点击run运行。这里提供一个简单的方法：在burp-loader-keygen.jar和Burpsuite同目录下创建一个bat文件，里面写入
java -Xbootclasspath/p:burp-loader-keygen.jar -jar burpsuite_pro_v2.0.05beta.jar，然后直接打开bat文件就可以打开了。（bat文件附带在压缩包内）
